    Loading leads

    Lead stages > Uploading leads
    Method upload.leads_history
    Description Loading leads
    Available to Agent, Client

    The method allows you to download lead history and transitions between stages. - For the client contact_ext_id we look for data on an already created contact using the method "Creating a contact" - By client_name, if you did not find it by contact_ext_id, create a new contact. - Using contact information contact_ext_id we are looking for all requests from the contact (call, request, chat, email) - By source we look for the created source through the method "Creating a CRM source" - Using employee_ext_id we look for data on an already created employee using the "Creating an employee" method. - By communication_id and communication_type we associate the lead with this request - If contact_ext_id and communication_id, communication_type were sent simultaneously, the link to the appeal will be communication_id and communication_type - We filter out requests that fall under the filter - the time between requesting and creating an entity is more than 30 days - It is possible to update contact information (contact_ext_id) or treatment (communication_id, communication_type) on the lead only with a shift in the stage modification date (modified_date_time, modified_stage_date_time). The link between the lead and the request will be redefined.

    If the data for one of the leads is filled in incorrectly, the request ends with an error and after correcting it, the entire request must be submitted again.

    Request parameters

    Title Type Required Description
    access_token string yes Authentication session key
    user_id number no Unique user identifier of the agent client on behalf of which the request is made
    Is required for the agent
    To obtain a list of client users, you must use the method "get.customer_users"
    ext_id string yes Lead ID in external system
    name string yes Lead name in external system
    emails array no List of emails left in the lead
    phones array no List of phone numbers left in the lead
    contact_ext_id string no Contact ID in external system
    Messengers
    messagers object no Information from the channel of other messengers
    id string no ID
    type string no Type
    profile string no Profile
    client_name string no Client name
    created_date_time iso8601 yes Lead creation date and time (YYYY-MM-DD hh:mm:ss)
    modified_stage_date_time iso8601 yes Date and time the lead moves to the current stage (YYYY-MM-DD hh:mm:ss)
    closed_date_time iso8601 no Lead closing date and time (YYYY-MM-DD hh:mm:ss)
    employee_ext_id string no Responsible employee for lead
    company_ext_id string no Related companies
    stage_ext_id string yes Lead Funnel Stage ID
    source string no Lead Source
    communication_id number no ID of the case to which the transaction is associated. Mandatory if communication_type is passed
    communication_type string no The type of treatment with which the transaction is associated. Available values ​​are chat,offline_message,call,goal,email. Mandatory if communication_id is sent.
    user_fields array no Custom fields added to the lead. "create.lead_user_field"
    ext_id string yes Transaction Custom Field External ID
    utm_campaign text no Campaign UTM tag.
    utm_content text no UTM tag Content.
    utm_medium text no Medium UTM tag.
    utm_source text no Source UTM tag.
    utm_term text no UTM tag Term.
    child_customer_id number no Identifier of the client within which the Transaction is created. Method for getting a list of clients: "get.customers"
