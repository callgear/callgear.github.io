    Loading sessions

    Loading sessions > Loading sessions
    Parameter Meaning
    Necessary components for working with the method "Data API for configuration", "Loading sessions from an external system"
    Method weight 1 point
    Maximum number of sessions in one request 50 pcs
    Maximum JSON size 512 KB
    IP address of the server that needs to be given access to download files 195.211.122.249
    Method upload.sessions
    Description Loading sessions for end-to-end analytics
    Available to Agent, Client

    Request parameters

    \
    Name Type Required Valid values Description
    access_token string yes Authentication session key
    user_id number no Unique user identifier of the agent client on behalf of which the request is made
    Required for the agent
    To obtain a list of client users, you must use the "get.customer_users" method
    ext_id string yes A unique session identifier in the external system.
    Duplicates are discarded when sessions are loaded into the database
    . To match ext_id and internal session id, use the method "get.visitor_sessions_report"
    site_id number yes Unique site identifier
    date_time iso8601yes YYYY-MM-DD hh:mm:ss Date and time of the session.
    The data will be adjusted taking into account the specified time zone in your personal account. The date must not be greater than the current time and less than 2011-01-01 00:00:00
    visitor_id number no Can only be from the same site_id Unique visitor identifier
    is_visitor_new boolean yes Is the visitor new
    duration number no start_time + duration must be no more than 23:59:59 date_time Session duration, value in seconds
    visitor_ip_address string yes Visitor IP address
    campaign_id number no Unique identifier of the advertising campaign. If not specified, we will try to determine the campaign using the passed parameters. If they could not determine, then the session is assigned to the advertising campaign c campaign_id = -1 Посетители без рекламной капании

    ### JSON response structure
<blockquote>The response comes as soon as the system has saved the session for further processing. The response contains <code>ext_id</code> of the transferred session and the <code>session_id</code> created on its basis in our system</blockquote>```json
{
  "jsonrpc": "2.0",
  "id": "number",
  "result": {
    "data": [
        {
            "ext_id": "string",
            "session_id": "number"
        }
    ]
  }
}
