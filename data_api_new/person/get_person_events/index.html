    Getting a list of all hits

    Retrieving a list of all person events for a period > Obtaining a list of all events of a person for a period
    Method get.person_events
    Description Obtaining a list of all events of a person for a period
    Available to Agent, Client
    Necessary components for working with the method Data API for configuration
    Method weight 1 point

    Request parameters

    Name Type Required Valid values Description
    access_token string yes Authentication session key
    user_id number no Unique user identifier of the agent client on behalf of which the request is made
    Is required for the agent
    To obtain a list of client users, you must use the method "get.customer_users"
    limit number no You cannot request more than 200 lines. Number of records to return. Default 50.
    offset number no Shift - determines from which record number to return limit records. Default 0.
    person_id enum yes Unique identifier of the person for whom general information is required
    date_from iso8601 yes YYYY-MM-DD hh:mm:ss Start date of main sample (limit 366 days)
    date_till iso8601 yes YYYY-MM-DD hh:mm:ss End date of the main sample
    fields array yes List of field mnemonics, separated by comma. For a list of available fields, see "Getting column nesting" . The order affects the construction of the tree.
    Filters used in the report
    filter object no It is permissible to use no more than 2 уровней nesting. See examples belowFilters applied in the report. The construction logic is similar to "Filtering criteria"
    filters array yes An expression can contain both simple filters and a filter tree.
    field string The field of the entity to which filtering will be applied (the list is predefined for the method
    operator enum Filtering operator. A list of all operators can be obtained in the section "Filtering operators"
    value string Value for the filtering operator. Optional field; if it is missing, it is considered empty.
    Sorting used in the report
    sort array no The sorting used in the report. The construction logic is similar to "Data sorting"

    Response parameters

    Name Type Valid values Filtering Sorting Default response Description
    site_domain_name string yes yes Internet site address. Without specifying the protocol - "http://" or "https://"
    campaign_name string yes yes Advertising campaign name
    call_direction string yes yes Call direction
    call_status string yes yes Sign that the call is lost or successful
    chat_status string yes yes Chat status
    date_time_creation iso8601 YYYY-MM-DD hh:mm:ss yes yes Event date and time
    previous_event_time_delta text yes yes Difference between date_time_creation current and previous event
    deal_current_stage string yes yes Current stage of the transaction
    lead_current_stage string yes yes Current lead stage
    offline_message_type string yes yes Application type
    lead_name string yes yes Lead
    deal_name string yes yes Deal
    goal_name string yes yes Target name
    event_type object yes yes Event type
