    Getting a list of all calls

    Getting a list of all calls > Getting a list of all calls
    Method get.mgf_calls
    Description Getting a list of all calls
    Available to Agent, Client
    Necessary components for working with the method Data API for configuration
    Method weight 1 point

    Request parameters

    method
    Name Type Required Valid values Description
    access_token string yes Authentication session key
    user_id number no Unique user identifier of the agent client on behalf of which the request is made
    Is required for the agent
    To obtain a list of client users, you must use the method "get.customer_users"
    limit number no Number of records to return. See "Paging Output"
    offset number no Shift, determines from which record number to return `"limit"` records. Default is "0". See "Paging Output"
    report_id number yes User report unique identifier. To get a list of user reports, you must use the "get.reports_list"
    date_from iso8601 yes YYYY-MM-DD hh:mm:ss Start date of main sample (limit 366 days)
    date_till iso8601 yes YYYY-MM-DD hh:mm:ss End date of the main sample
    fields array no List of custom column identifiers, separated by comma. Required to obtain data. If the parameter is empty or not passed, the response will be empty. For the available list of columns, see "Getting Column Nesting" . The order affects the answer.
    Filters used in the report
    filter object no It is permissible to use no more than 2 уровней nesting. See examples below Filters applied in the report. The construction logic is similar to "Filtering criteria"
    filters array yes An expression can contain both simple filters and a filter tree. For an available list of filters for a report with their possible values, see section "Getting a list of possible filtering conditions for a report" .
    field string Field of the entity to which filtering will be applied (the list is predefined for the method)
    operator enum Filtering operator. A list of all operators can be obtained in the section "Filtering operators"
    value string Value for the filtering operator. Optional field; if it is missing, it is considered empty.
    Sorting used in the report
    sort array no The sorting used in the report. The construction logic is similar to "Data sorting"

    Response parameters

    Name Type Valid values Filtering Sorting Default response Description
    communication_id string yes yes yes Unique Call ID. Call session ID
    call_start_date iso8601 YYYY-MM-DD yes yes yes Date of call
    call_start_time iso8601 hh:mm:ss yes yes yes Call time
    contact_phone_number string yes yes yes The contact number from which the call was received in the case of an incoming call or the contact number to which the call was made in the case of an outgoing call.
    call_direction object yes yes yes Call direction
    call_status object yes yes yes Call status. Sign that the call is lost or successful
    call_type object yes yes yes Call type
    virtual_phone_number string yes yes yes Virtual number
    employees array yes Employees who participated in the call
    call_total_duration duration yes yes yes Call duration
    call_await_talk_duration duration yes yes yes Waiting time for conversation to start
    call_talk_duration duration yes yes yes Conversation duration
    call_is_internal object yes yes yes External/internal call
    call_finish_reason object yes yes yes Reason for calling end
    employee_rating numeric yes yes yes Call rating
    call_records array yes yes Unique identifier of the link to the recorded conversation.
    communication_comment string yes yes yes Comment
    communication_comment_author string yes yes yes Comment author
    received_call string yes yes yes Who received/made the call. The last operation in an incoming missed call or the first answer/first employee in an outgoing call
    first_employee_for_internal_call string yes yes yes Call initiator. For internal calls only
    second_employee_for_internal_call string yes yes yes Who did you call? For internal calls only
    call_finish_date_time iso8601 YYYY-MM-DD hh:mm:ss yes yes yes Call end time
    coaches array yes yes yes Name of the employee who was a trainer for the employee who participated in the call
    is_coach_handled_call object yes yes yes Trainer connection status. Shows that the Coach was involved in one of the calls
    virtual_number_comment string yes yes Note on virtual number
    is_blacklisted_phone object yes yes yes Is the subscriber's number blacklisted
