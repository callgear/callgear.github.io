    Instructions for downloading transactions from CRM

    Sales Funnels and Deals

    Intro

    Name Description
    Deal A sales card that goes through the stages of the sales funnel.
    Contact Customer card from your CRM system.
    Sales funnel It consists of the stages that a transaction goes through from creation to successful completion (for example, payment).
    Stage The part of the sales funnel that a deal takes through a company's sales process.
    Responsible employee An employee in CRM responsible for a specific transaction.
    To analyze the effectiveness of advertising campaigns before the transaction and profit in analytical reports you need to upload transactions from your CRM to your CallGear personal account.
    You can do this by connecting “End-to-end analytics” and setting up integration with CRM using ready-made solutions ( "Bitrix24" , "amoCRM" or "MIS Infoclinic" ), where the transfer of transactions to CallGear is implemented.
    If you have another CRM system, then you can use API and configure the integration yourself.

    To set up integration via API you need:

    1. Create a CRM structure in CallGear
      • Pass the structure of each sales funnel
      • Submit a list of employees and persons responsible for each transaction
      • Send contact list
    2. Load transaction history
    3. Configure end-to-end analytics
    4. Set up the necessary data for analysis in analytic reports


    Using a test client as an example, let's look at how to transfer transactions and other necessary data from your CRM.

    Example Transfer of transactions and other data from CRM
    Description Name of the funnel: "Main funnel".
    Sales funnel structure:
    • stage 1 - “New deal”
    • stage 2 - "Preparation of documents"
    • stage 3 - “Invoice for prepayment”
    • stage 4 - "Final score"
    • successful stage - “Successfully implemented”
    • failed stage - “Deal failed”
    Funnel elements:
    2 responsible employees
    3 contacts
    3 trades:
    • deal 1: went through all stages and closed with a sale
    • deal 2: passed the first stage and failed
    • transaction 3: reached the “Invoice for prepayment” stage
    Each transaction has a contact and a responsible employee.

    1. Transferring the structure of your CRM to CallGear

    1.1 Creating a sales funnel structure:

    Method create.sales_funnel
    Description Creating a sales funnel and its stages
    Available to Agent, Client
    В запросе "Порядковый номер этапа" указан как 10, 20, 30, 40.

    Это сделано, для того чтобы в будущем вы смогли добавить промежуточные этапы без редактирования всей воронки. Допустим, в CRM поменялась структура воронки, и добавился этап "Подписание документов" между этапами "Подготовка Документов" и "Счет на предоплату". Вы легко можете добавить этап "Подписание документов" с порядковым номером от 21 до 29. Но лучше 25.

    Максимальный номер этапа = 10000

    JSON request structure```json
{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "create.sales_funnel",
    "params": {
        "access_token": "${#Project#auth_prod_4735}",
        "ext_id": "1",
        "name": "Основная воронка",
        "stages": {
            "success": {
                "ext_id": "1000",
                "name": "Успешно реализовано"
            },
            "failed": {
                "ext_id": "10001",
                "name": "Сделка провалена"
            },
            "in_process": [
                {
                    "ext_id": "1",
                    "name": "Новая сделка",
                    "order": 10
                },
                {
                    "ext_id": "2",
                    "name": "Подготовка Документов",
                    "order": 20
                },
                {
                    "ext_id": "3",
                    "name": "Счет на предоплату",
                    "order": 30
                },
                {
                    "ext_id": "4",
                    "name": "Финальный счет",
                    "order": 40
                }
            ]
        }
    }
}

    The response will return the ID of the created funnel.

    ###<b>2. Loading transaction history</b>

To build correct analytics, it is necessary to transmit all the stages that the transaction has gone through, and not just its current stage (at the time of upload).

<div>This can be done in two ways:</div>
<ol>
    <li> Transmit an update on a deal to CRM using a trigger, i.e. upload it to CallGear when one of the events occurs:
        <ul>
            <li>Creating a deal</li>
            <li>Transfer to another stage</li>
            <li>Updating one of the parameters: contact, owner, transaction amount</li>
        </ul></li>
    <li>
        <div>
                Once per period (hour, day, week) download the entire history for all transactions at once.
                For each transaction, transfer all completed stages and the time of transition to each stage.
        </div>
        <div> <br>All transferred transactions are displayed in the "End-to-End Analytics" report cumulatively (that is, all stages that the transaction has passed through are displayed, not just the current one). </div>
        <div> <br>If any of the intermediate stages has not been transferred, the system will automatically create it with the flag is_generated = True,
        this is necessary for correct display in analytical reports. Stage indicators take into account transactions that have passed through this stage. </div>
    </li>
</ol>```sh
Отдельного метода по созданию и обновлению сделки у нас нет.
Сделка создается при передачи первой записи по ней, и обновляется
с каждым новым вызовом метода.
